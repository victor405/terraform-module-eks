# USED
- **aws_eks_access_entry** - Defines access entries for an EKS cluster, specifying user or role permissions.
- **aws_eks_access_policy_association** - Associates IAM policies with an EKS cluster to manage access controls.
- **aws_eks_addon** - Manages EKS cluster add-ons for extended functionalities (e.g., CoreDNS).
- **aws_eks_cluster** - Creates and configures an Amazon EKS cluster.
- **aws_eks_fargate_profile** - Configures EKS to run pods on AWS Fargate (serverless computing).
- **aws_eks_identity_provider_config** - Integrates identity providers (e.g., OIDC) for authentication to EKS.
- **aws_eks_node_group** - Manages a set of EC2 instances as worker nodes in an EKS cluster.
- **aws_eks_pod_identity_association** - Associates IAM roles with specific EKS pods for fine-grained access control.
